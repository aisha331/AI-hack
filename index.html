ğŸ“… Day 2 â€” "Reverse Engineering and Rate Limits ğŸ”§ğŸš«"

Ollie

"I spent the day diving into the CMD+K feature, trying to extend our existing autocomplete and quick fix infrastructure. Started by figuring out how to get the right data flowing into CMD+K, which sent me down the MCP rabbit hole. That turned out to be a dead end pretty quickly - Mintlify just doesn't expose the endpoints we'd need to make a chat feature work properly."

"Next attempt was using llms.txt from our docs to feed in all the APL info. Worked okay, but it was pretty expensive and slow."

"After both of those failed attempts I switched gears to focus on prompt engineering instead. Feed it a bunch of sample queries and fine-tuned instructions for different scenarios. The cool thing was that when combined with the LLM's built-in knowledge of KQL and our dataset structure, it actually performed way better than expected for APL work."

Gabriel

"I focused on reverse-engineering how Mintlify's MCP works under the hood. Spent hours in the Chrome DevTools trying to figure out their API structure and authentication flow. Found some interesting endpoints but kept hitting rate limits."

"Eventually discovered their token-based system had some gaps we could potentially work with, but the documentation was pretty limited. Started building a small adapter to translate between their API format and what our CMD+K feature expects."

Mark

Implementation and Infrastructure Setup

"On the second day, I implemented the training pipeline locally and worked with Conor on setting up infrastructure on the dev environment to host and run the model. I chose phi-4-instructed for training, which can be "instructed" to follow specific prompts or tasks effectively, which I believe makes it better to solve contextual problems such as understanding what exactly "APL" is and is not."

"Initially, I hit a roadblock when my M4 Mac couldn't handle the RAM demands, even with a small sample. To overcome this, I shifted training to AWS, with tons of GPU and vCPU resources. I also developed a UI and Python code to support iterative training, incorporating human feedback to refine the model's suggestion quality, through UI."

Aisha

"Claude was successfully hosted on AWS to address privacy considerations. Spent the day extensively testing various prompts and tuning parameters to optimize LLM output accuracy."

"Gained a deeper understanding of how MCP clients operate and experimented with MCP servers via CLI to evaluate potential integration with Axiom."

"Additionally, explored multiple approaches to fine-tuning models, which I plan to investigate further over the weekend."
